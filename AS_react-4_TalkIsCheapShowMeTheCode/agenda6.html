Agenda

  • Monolith Architecture: Monolith means composed all in one piece. The Monolithic application describes a single-tiered software application in which different components combined into a single program from a single platform.

    1. 1 Project = API's written + UI + Authentication Code + Database Connectivity + Sending SMS
    2. Despite having different components/modules/services, the application is built and deployed as one Application for all platforms (i.e. desktop, mobile and tablet)

  • MicroService Architecture:

    1. We have different service for different job. Like: Backend Service, UI Service, UI Project, Authentication Service, Service connects to Database, Service to send SMS, Service for Email notifications.
    2. Theses all services combines together forms a big app.
    3. We have all as separate project => Separation of Concerns.
    4. It follows Single responsibility Principle where each service has its own job, no-one interfere in it.
    5. How these services interact with each other to work application as a single project? =>
      1. They make call to different port/url. Like UI call an API.
    6. How it deploy?
      1. Each service run on their own specific port. We deploy each service on different port.
      2. All ports can be mapped to domain name. API -> abc.com/api, UI -> abc.com

  • How React Application, make connection with different microservices.

  • How UI Application, fetch data from backend?

    1. Approach 1 : As soon as page loads - make an API call, when we get data, render it into UI. If API call take 500ms, page will wait & after 500ms it will load UI.
    2. Approach 2 : As soon as page loads - we will render UI then make API call, after completion of API data fetch, We will rerender (React render cycle is very fast.) our application with new data.
    3. In React, we will always using Approach 2, better Approach with better UX.
      1. No feel like page frozen.
      2. Atleast we see UI skeleton.
